看博客搭博客——Hexo + GitHub Pages

写在前面

准大四生，个人认为应当做个博客记录一下自己的学习以及生活，权当笔记及回忆。
第一次开始搭博客，反复折腾了好几天，看别人写的搭建博客教程来搭建自己的博客。
在此简单记录下自己的搭建博客的过程！

关于GitHub Pages以及Hexo

GitHub Pages

服务器在国内，不用翻墙的就能访问的GitHub，对程序员来说就是福音。再具体的的就要Google了。GitHub设计了Pages功能，允许用户自定义项目首页，用来替代默认的源码列表。所以，github Pages可以被认为是用户编写的、托管在github上的静态网页。由此可用来做博客。
附上简介：http://blog.ssyog.com/blog/jekyll/github-pages-intro.html

hexo

Hexo是一个简单、快速、强大的静态博客框架,出自台湾大学生tommy351之手。下面是Hexo官网强调Hexo的四大特点：

  • 有300M免费空间，资料自己管理，保存可靠；
  • 享受 GitHub 的便利，上面有很多大牛，眼界会开阔很多；
  • 可以使用很多现成的博客框架，搭建过程简单快速.

环境配置及部署

###安装Nodejs（必须）
作用：用来生成静态页面的, win\mac\linux都有相关版本自行到官网下载。
Nodejs可以选择稳定版本4.5.0，下载下来选择安装路径一路Next即可。

安装Hexo

打开git shell命令终端：

$ cd d:/hexo
$ npm install hexo-cli -g # 安装hexo
$ hexo init blog # 在d:/hexo/目录下初始化
$ cd blog
$ npm install # npm将会自动安装需要的组件，只需要等待npm操作即可
$ hexo g # 或者hexo generate
$ hexo s # 或者hexo server，可以在http://localhost:4000/ 查看

至此，打开http://localhost:4000/ 已经可以看到一篇内置的blog。

另外，提一下Hexo的常用命令
1.hexo generate (hexo g) 生成静态文件，会在当前目录下生成一个新的叫做public的文件夹
2.hexo server (hexo s) 启动本地web服务，用于博客的预览
3.hexo deploy (hexo d) 部署播客到远端（比如github, heroku等平台）
4.hexo new “postName” #新建文章
5.hexo new page “pageName” #新建页面

常用简写

$ hexo n == hexo new
$ hexo g == hexo generate
$ hexo s == hexo server
$ hexo d == hexo deploy

Hexo全局配置

使用hexo deploy部署

hexo deploy可以部署到很多平台，具体可以参考这个链接. 如果部署到github，进入根目录d:/hexo/blog(即hexo初始化的目录)，找到_config.yml文件修改配置：

deploy:
type: git
repo: git@github.com:Thinking-Stone/Thinking-Stone.github.io.git
branch: master

一些解释：repo中的Thinking-Stone为自己在github上的名称
然后在命令行中执行

hexo d

注意：需要提前安装扩展：
<$ npm install hexo-deployer-git –save

随后按提示输入创建SSH keys时的密码，提示成功后可在https://thinking-stone.github.io/访问网站

更换主题

附上Hexo的主题,下载所喜欢的主题文件，解压到d:/hexo/blog根目录下的themes文件夹。然后在_config.yml文件中修改：

Extensions

Plugins: http://hexo.io/plugins/

Themes: http://hexo.io/themes/

theme: landscape //themes文件夹中对应文件夹的名称

在git shell终端中进入d:/hexo/blog（即根目录）目录中重新

hexo g
hexo d 或者 hexo s
之后就可在https://thinking-stone.github.io/或者http://localhost:4000/看到效果
